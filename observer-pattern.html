Observer Pattern

当一个对象的行为依赖于另一个对象的状态时，观察者模式就相当有用。 若不使用观察者模式提供的通用结构，则只能在另一个线程中不断监听对象所依赖的状态

经典结构图：

ISubject接口 ···java public interface ISubject { public void attach(IObserver o); public void deattach(IObserver o); public void inform(); }

IObserver接口 ```java public interface IObserver { public void update(Event evt); }

ConcreteSubject实现 ```java import java.util.*;

public class ConcreteSubject implements ISubject{ private List observers = new ArrayList(); @Override public void attach(IObserver o) { observers.add(o); }

@Override
public void deattach(IObserver o) {
    if(observers.contains(o))
        observers.remove(o);
}

@Override
public void inform() {
    Event e = new Event();
    for(IObserver o : observers) {
        o.update(e);
    }
}

}

ConcreteObserver实现 ```java public class ConcreteObserver implements IObserver{

private int id;

public ConcreteObserver(int id) {
    this.id = id;
}
@Override
public void update(Event evt) {
    System.out.println("Observer " + id + " Get Information from Subject.");
}

}

测试主类 ```java public class Main { public static void main(String[] args) { int n = 10; ISubject cs = new ConcreteSubject(); for(int i = 0; i < n; i++) { IObserver observer = new ConcreteObserver(i); cs.attach(observer); } cs.inform(); } }
