window相关的概念

将无限流切割为有限流的方式，将流数据分发到有限大小的bucket

window类型 - 时间窗口(Time Window) - 滚动时间窗口 - 滑动时间窗口 - 会话窗口 - 计数窗口(Count Window) - 滚动计数窗口 - 滑动计数窗口

滚动窗口 Tumbling Windows

  1. 固定窗口长度
  2. 无缝连接, 窗口之间没有重叠
  3. 时间点是左闭右开 例如[9:00, 10:00), [10:00, 11:00), ...

滑动窗口 Sliding Windows

  1. 滑动窗口有固定的窗口长度和滑动步长
  2. 窗口之间可以重叠

会话窗口 Session Windows

1. 要指定一个session gap, 只要两个数据到来时间超过session gap, 就生成新的窗口

Java CompletableFuture

简介

CompletableFuture是java8引入的一种异步执行工具，它同时实现了Future接口和CompletionStage接口，前者使其具备异步执行的能力，而后者则是让当前的异步执行和其他异步执行联系起来

基本用法

用作Future

Future表示一个异步执行结果

public Future<String> calculateAsync() throws InterruptedException {
    // 无参构造
    CompletableFuture<String> completableFuture 
      = new CompletableFuture<>();

    Executors.newCachedThreadPool().submit(() -> {
        Thread.sleep(500);
        // complete方法的入参即返回结果
        completableFuture.complete("Hello");
        return null;
    });

    return completableFuture;
}

Future<String> completableFuture = calculateAsync();

// 其他逻辑

String result = completableFuture.get();
assertEquals("Hello", result);

编写React的注意事项

  1. 外部引入的组件直接使用文件名(不需要增加文件后缀)
import React from 'react';
import ReactDOM from 'react-dom';
  1. 引入自己项目的资源必须以./或../开头
import './index.css';
import App from './App';
import * as serviceWorker from './serviceWorker';
  1. 自定义组件的自定义函数使用箭头函数
addComment = (comment) => {
    const {comments} = this.state
    comments.unshift(comment)
    this.setState(comments)
}
  1. 自定义组件的props类型规范使用static语法
static propTypes = {
    addComment: PropTypes.func.isRequired
}
